# DCYFR brand-mark assets

Owned, hand-authored vector assets for the DCYFR mark. Generated by
[`scripts/brand/build-crystal-mark.mjs`](../../../scripts/brand/build-crystal-mark.mjs)
— edit the generator (or `src/lib/logo-config.ts`) and re-run, never hand-edit the SVGs.

> **Status: awaiting brand-owner (Drew) approval of the vector.** Do not ship the
> faceted mark to favicon / OG / production surfaces until approved. Provenance +
> decision: [`openspec/changes/dcyfr-crystal-brand-mark/`](../../../openspec/changes/dcyfr-crystal-brand-mark/).

## The mark

The **"evolve" crystal-facet treatment** of the four-point sparkle (ratified
2026-06-05): the *exact same silhouette* as the current mark, divided into 8 flat
facets (2 per arm) by straight internal seams. It is **not** a replacement and **not**
the "D" monogram — it keeps the sparkle's soft concave outer curves; the crystal read
comes from the straight spines + the flat-tone facet split.

The 3D crystal-sparkle render (dcyfr-labs#695) remains **hero/marketing art only** —
it is not this mark.

## Files

| File | Use |

| `dcyfr-mark-crystal-dark.svg` | Faceted mark on **dark** surfaces — **≥64px only** |
| `dcyfr-mark-crystal-light.svg` | Faceted mark on **light** surfaces — **≥64px only** |

| `dcyfr-lockup-horizontal-{dark,light,mono}.svg` | Mark + DCYFR wordmark, horizontal |
| `dcyfr-lockup-stacked-{dark,light,mono}.svg` | Mark + DCYFR wordmark, stacked |

## Size rule (load-bearing)

The faceted mark **must only be used at ≥64px** (large logo, OG/social, hero). Below
~32px the white/blue facets blur into a muddy blob — proven by the reduction test in
the exploration. **The favicon and all small/monochrome uses take the flat
silhouette** (`dcyfr-mark-flat.svg` / the existing `LOGO_PATH`). This is intentional,
not a limitation.

In the app, render the faceted mark via `<CrystalMark surface="dark|light" />`
(`src/components/common/crystal-mark.tsx`) and the flat mark via `<Logo />`. Both share
one geometry source: `src/lib/logo-config.ts`.

## Palette

Deep navy `#020617`, electric blue `#2563eb`, crystal cyan `#38bdf8` / `#0ea5e9`,
white `#f9fafb` — the dcyfr identity (electric blue/cyan, never purple). Mirrored in
`design-tokens.ts` → `BRAND_MARK` and `logo-config.ts` → `CRYSTAL_MARK_COLORS`.

## Wordmark

The lockups set **DCYFR** in **Alegreya SemiBold** (the brand logo face, `--font-serif`).
The lockup SVGs use live `<text>`; **production should outline the wordmark to paths**
so the assets are font-independent. The mark itself is already pure outlined vector.
